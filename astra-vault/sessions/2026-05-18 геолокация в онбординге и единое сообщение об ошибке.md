# Сессия 2026-05-18 — геолокация в онбординге

## Цель

Починить кнопку «📍 Отправить мою геолокацию» в онбординге (место рождения + «где живёшь»), убрать «Как место рождения», обеспечить ручной ввод и понятное поведение на телефоне и Desktop.

## Сделано

- **Единый шаг места** для `birth_place_query` и `notification_place_query`: reply-клавиатура с `request_location` + ручной ввод.
- **Убрана** inline-кнопка «📍 Как место рождения» и callback `place:notif_same`.
- **Исправлены конфликты хендлеров**: текст кнопки больше не уходит в поиск города; `F.location` и `StateFilter` в `places.py`.
- **Телефон**: геолокация работает → ближайший Place из GeoNames → подтверждение.
- **Desktop**: `request_location` шлёт текст, не координаты → единый ответ пользователю (см. ниже).
- **Web App** для браузерной геолокации (`/telegram/webapp/location`, `WEBAPP_BASE_URL`) — добавлен, затем убрана отдельная кнопка «💻 Геолокация (компьютер)»; Web App остаётся в коде для `web_app_data`.
- **Единое сообщение об ошибке геолокации**: `GEOLOCATION_FAILED_TEXT` → `reply_geolocation_failed()` во всех сбоях.
- **`/start restart`** — повторная регистрация без сброса БД вручную.
- Тесты: 22 pytest (в т.ч. `test_webapp.py`).
