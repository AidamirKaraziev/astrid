### Домашний сервер (deadtiger)

| | |
|---|---|
| **CPU** | Intel Core i7-4810MQ @ 2.80 GHz (Haswell 2014, 4 ядра / 8 потоков) |
| **RAM** | 8 GB DDR3-1866, одна плашка, **одноканальный режим** |
| **Хранилище** | 100 GB HDD (не SSD) |
| **GPU** | GTX 765M, 2 GB VRAM (Kepler) — **бесполезна для LLM**: compute capability 3.0, Ollama/CUDA требуют 5.0+ |
| **ОС** | Ubuntu 24, без GUI |

**Выводы для решений:**

- LLM — только CPU-инференс; бюджет RAM под модель ≈ 4–5 GB (Docker-стек postgres+redis+rabbitmq+api+worker съедает ~2–3 GB). Реалистичный потолок — модели ~4B параметров в Q4 (например `gemma4:e2b`); 7B+ — впритык/swap.
- Одноканальная DDR3 → низкая пропускная способность памяти, инференс медленный (~2–5 t/s); генерацию держать в фоновых очередях, не в request/response.
- HDD: холодная загрузка модели в RAM — десятки секунд; держать модель тёплой (`keep_alive`).
- Память — главный кандидат на апгрейд: вторая плашка DDR3 → двухканал ≈ ×1.5–2 к скорости инференса дёшево.
