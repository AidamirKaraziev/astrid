# На Desktop request_location приходит как текст, а не location

## Контекст

Кнопка `KeyboardButton(request_location=True)` в Telegram на **iOS/Android** отправляет сообщение с `message.location`. На **Telegram Desktop** часто приходит только **текст кнопки** — координат нет.

## Решение

- Не полагаться на Desktop для `request_location`.
- При тексте кнопки геолокации (и прочих сбоях) — одно сообщение: «Не смог получить твою геопозицию, введи название в ручную» (`reply_geolocation_failed` в `places.py`).
- Ручной ввод названия города всегда доступен на том же шаге.
- Опционально: Web App с `navigator.geolocation` + `web_app_data` (нужен HTTPS и домен в BotFather); отдельная кнопка для Desktop убрана по UX.

## Альтернативы

- Только инструкция 📎 → Геопозиция — отвергнуто в пользу короткого сообщения об ошибке.
- Определение Desktop по эвристике — ненадёжно.
