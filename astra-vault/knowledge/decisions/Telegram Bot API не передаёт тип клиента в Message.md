# Telegram Bot API не передаёт тип клиента в Message

## Контекст

Нужно было показывать разный UX на телефоне и Desktop (разные кнопки геолокации).

## Решение

В объектах `User` / `Message` **нет** поля device/platform/client. Не строить логику на «угадывании» устройства по типу апдейта.

**Исключение:** при открытии **Web App** в JS доступно `Telegram.WebApp.platform` (`ios`, `android`, `tdesktop`, `web`, …) — можно передать в бот через `sendData`, если понадобится аналитика.

## Практика в Astra

Одна reply-клавиатура для всех; Desktop при нажатии геокнопки получает [[единое сообщение когда геопозицию не удалось получить или обработать]].
