### 3. Нагрузочное тестирование (PERF-1) — scope на утверждение

Предлагаемый минимальный scope для deadtiger (i7-4810MQ, 8 GB RAM):

| Сценарий | Что симулируем | Метрики |
|----------|----------------|---------|
| **Burst 09:00** | N пользователей одновременно запрашивают прогноз в одну минуту | очередь RabbitMQ, p95 latency, OOM |
| **Sustained** | M прогнозов подряд без паузы | CPU/RAM, время одной генерации |
| **API health** | RPS на `/health`, `/v1/predictions/today` | p99, ошибки 5xx |
| **Scheduler** | 100+ юзеров в PG, tick scheduler | время одного цикла, пропуски |

**Предлагаемые пороги (черновик — поправь):**

- p95 генерации одного прогноза: **≤ 180 сек** (deadtiger)
- OOM / kill контейнера: **0** при N ≤ **10** одновременных генераций
- Очередь RabbitMQ: без unbounded growth при burst 20 юзеров

**Инструмент:** скрипт на Python (async httpx + publish в RabbitMQ) или k6/Locust — решить при реализации.

**Утвердить:** пороги N/M, инструмент, запускать до или после беты.
